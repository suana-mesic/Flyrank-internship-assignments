## How to add the next case

The portfolio is a single `index.html` file. Adding a case means copying the BookVerse block in the Work section and replacing the content. The structure is always the same three beats from FL-02:

1. **The problem** — what was broken, slow, or missing.
2. **What I did** — the specific technical decision I made.
3. **The result** — what changed, with evidence (a number, a test, a before/after).

Steps:

1. Open `index.html` in VS Code.
2. Find the `<!-- ================= WORK ================= -->` section.
3. After the BookVerse `</figure>` and repo link, add an `<hr>` and paste a copy of the BookVerse block.
4. Replace: `<h2>` title, `.case-lead` paragraph, `.stack` tech tags, the three `.beat` divs, the code block or image, and the repo link.
5. Push to `suana-mesic.github.io` — GitHub Pages publishes automatically.

Total time per case: ~1 hour if the project is already finished and I know which decision to highlight.
